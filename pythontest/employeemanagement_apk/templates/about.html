Athimet Aiewcharoen Task

Assignment: Develop an Employee Management Application

Objective

  1. Project Setup:
    • Create a new Django or Flask project with two environments: development and production.
  2. Models:
    • The employee management application should have four models: Employee, Position, Department, and Status.
    • Employee model: Contains employee information such as:
      • Name (Text)
      • Address (Text)
      • Manager (Y/N)
      • Status (Status model)
      • Image (JPG)
    • Position model: Contains position name (Text) and salary (Number).
    • Department model: Contains department name (Text) and manager (Employee model).
    • Status model: Contains the current status of the employee (e.g., in recruitment process, waiting for onboarding, in probation period, normal, and resigned).
  3. User Authentication:
    • Add a new login account with a normal user role using Django's built-in authentication or Flask-Login.
  4. CRUD Operations:
    • Create: Add new entries to the Employee, Position, Department, and Status models.
    • Read: Display lists of Employee, Position, Department, and Status models.
    • Update: Edit existing entries in the Employee, Position, Department, and Status models.
    • Delete: Remove entries from the Employee, Position, Department, and Status models.
    • Ensure that related data in other models are handled appropriately.
  5. Advanced Queries:
    • Implement filtering and searching functionality for Employees by Position, Department, and Status.
  6. API Development:
    • Create a CRUD RESTful API using Django, Flask, or FastAPI for the Employee, Position, and Department models.
  7. Testing:
    • Write unit tests and integration tests for the models and API endpoints.
